# Contributing

Thanks for your interest in improving MacBook Pro Max System Monitor.

## Before You Start

- Open an issue for feature ideas, large refactors, protocol changes, or UI redesigns.
- Keep changes focused and easy to review.
- Include setup notes when your change affects installation or local development.
- Avoid committing secrets, device identifiers, local network addresses, or signing keys.

## Development Expectations

When the application source is added, contributors should:

- Run the relevant Mac agent and Android app checks before opening a pull request.
- Add or update tests for behavior changes where practical.
- Update documentation for setup, security, configuration, or user-visible behavior changes.
- Keep telemetry collection transparent and minimal.

## Pull Requests

Pull requests should include:

- What changed.
- Why it changed.
- How it was tested.
- Screenshots or screen recordings for Android dashboard changes.

## Code Style

Follow the existing style in each component. If a component has a formatter or linter, run it before submitting.
